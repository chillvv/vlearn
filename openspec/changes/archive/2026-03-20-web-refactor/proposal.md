## Why

当前 Web 端项目在数据层面存在大量硬编码的假数据（如知识宇宙、概念胶囊、错题列表等），并未真实连接到 Supabase 后端，导致核心功能（错题本和知识库）无法形成闭环。同时，UI 布局和排版显得杂乱，没有突出“英语”和“编程”两个核心学科的特质。为了让应用真正可用并提升用户体验，需要进行一次全面的前端重构，打通真实数据，精简学科，并对不完善的 UI 和功能进行修复与优化。

## What Changes

- **数据真实化**：移除前端所有的 mock 数据，所有关于错题、弱点、知识树的数据全部对接 Supabase 真实接口。如果后端没有数据，则通过真实交互（录入）来产生，而不是显示假页面。
- **学科精简**：移除数学、物理、化学等不需要的学科，全站（包括录入、仪表盘、知识库等）仅保留并专注“英语”和“编程”两个学科。
- **功能修复与优化**：修复目前残缺不全或无法正常使用的功能逻辑（例如知识宇宙的层级展开、专项训练的真实题目拉取、仪表盘的数据统计准确性）。
- **UI 优化（渐进式）**：对现有错乱的排版进行优化和修复，提升信息的展示清晰度。**注意**：如果是涉及到页面结构的彻底大重构（变形），会先出具 UI 设计草图或与用户沟通确认后再执行；如果只是优化残缺排版，则直接在现有结构上修复。

## Capabilities

### New Capabilities

- `real-data-integration`: 错题本与知识库全面对接 Supabase 真实数据读写，移除 Mock。
- `subject-focus`: 全局学科过滤机制，仅专注“英语”和“编程”。

### Modified Capabilities

- `dashboard-optimization`: 仪表盘数据真实化与排版优化。
- `knowledge-base-optimization`: 知识宇宙与概念胶囊的真实层级渲染与展示修复。
- `training-flow-fix`: 专项训练流程对接真实题目与 AI 接口修复。

## Impact

- **前端数据层**：大量 `mockData` 将被删除，全面替换为 `src/app/lib/api.ts` 中的异步调用。
- **UI 组件层**：部分页面的排版代码将被调整以适应真实数据的加载状态（Loading, Empty States）和更清爽的视觉呈现。
- **数据库**：可能需要在 Supabase 中预置部分“英语”和“编程”的真实基础知识节点（Knowledge Nodes），以避免初次登录时知识宇宙一片空白。
