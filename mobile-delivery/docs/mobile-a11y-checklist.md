# AIWeb Mobile Accessibility Checklist

## 1. 标准目标

- 目标等级：WCAG 2.2 AA。
- 覆盖平台：iOS VoiceOver、Android TalkBack。
- 最小触控：44x44dp。
- 正文与背景对比度：≥ 4.5:1。

## 2. 设计走查表

| 检查项 | 要求 | 结果 |
| --- | --- | --- |
| 文本对比度 | 正文 ≥ 4.5:1，图标 / 大字 ≥ 3:1 | 待走查 |
| 焦点顺序 | 顶部标题 → 内容 → CTA → 次级信息 | 待走查 |
| 按钮触控 | 所有按钮触控面积 ≥ 44dp | 待走查 |
| 颜色依赖 | 状态不只靠颜色表达 | 待走查 |
| 动效降级 | 支持减少动态效果 | 待走查 |
| 表单提示 | 错误提示与标签完整朗读 | 待走查 |
| 图片 / 图表 | 有语义说明或 alt 文案 | 待走查 |
| Bottom Sheet | 打开后焦点锁定，关闭返回触发源 | 待走查 |

## 3. 语音朗读细则

- 录题页：
  - 朗读顺序为标题 → 当前来源（拍照 / 相册）→ 上传按钮 → 草稿说明。
- 复习页：
  - 题干 → 选项 → 当前掌握等级 → 提交按钮 → 下一次复习时间。
- 统计页：
  - 指标名称 → 当前数值 → 趋势说明；图表需有等价文案摘要。

## 4. 焦点顺序细则

- 登录页：品牌 → 标题 → 说明 → 主登录按钮 → 生物识别按钮 → 补充说明。
- Dashboard：页面标题 → 关键 KPI 卡 → 功能卡片 → 底部导航。
- Bottom Sheet：标题 → 主内容 → 主按钮 → 次按钮 → 关闭按钮。

## 5. 验证步骤

1. 开启 VoiceOver / TalkBack。
2. 验证所有主路径可通过线性焦点完成。
3. 验证不看屏幕也能完成登录、录题、复习、提交。
4. 验证系统字体调大到 200% 不出现遮挡。
5. 验证深色模式与 OLED 模式下对比度仍达标。

## 6. 自动化建议

- 将无障碍命名纳入组件验收项。
- 在视觉回归之外补一套 `accessibilityLabel` 快照校验。
- 真机走查至少覆盖：
  - iPhone 13 / 15 Pro Max
  - iPhone SE
  - 小米 14
  - OPPO Find N 折叠
