## ADDED Requirements

### Requirement: 草稿确认后才能入库
系统 MUST 在用户录入错题后先生成结构化草稿确认视图，用户确认后才允许写入错题库。

#### Scenario: 用户确认后保存
- **WHEN** 用户在草稿确认页点击“准确无误，存入错题库”
- **THEN** 系统将当前草稿写入错题库并反馈保存成功

#### Scenario: 用户未确认不入库
- **WHEN** 用户尚未执行确认保存动作
- **THEN** 系统不得将草稿写入错题库

### Requirement: 草稿可被对话指令修订
系统 MUST 支持用户在草稿确认页输入修订指令，AI 返回修订后的完整 JSON 草稿并替换当前视图。

#### Scenario: 修订后即时更新页面
- **WHEN** 用户发送“只保留核心步骤”等修订指令
- **THEN** 页面应以新 JSON 重新渲染所有可见模块
