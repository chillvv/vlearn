# 📊 二维数组与数组指针

## 一、先理解二维数组

### 1. 二维数组的内存布局

```c
int a[2][3] = {
    {1, 2, 3},   // 第0行
    {4, 5, 6}    // 第1行
};
```

**内存图解**：
```
逻辑视图（2行3列）：
        列0    列1    列2
行0     1      2      3
行1     4      5      6

内存视图（连续存储，按行存储）：
地址:   1000  1004  1008  1012  1016  1020
        +-----+-----+-----+-----+-----+-----+
值:     |  1  |  2  |  3  |  4  |  5  |  6  |
        +-----+-----+-----+-----+-----+-----+
        ↑     ↑     ↑     ↑     ↑     ↑
       [0,0] [0,1] [0,2] [1,0] [1,1] [1,2]
```

---

### 2. 二维数组的"三层地址"概念 ⭐⭐⭐

**核心理解**：二维数组名可以理解为"数组的数组"

```c
int a[2][3];
```

| 表达式 | 含义 | 类型 | 值 |
|--------|------|------|-----|
| `a` | 整个数组的首地址 | `int (*)[3]`（行指针） | 1000 |
| `a[0]` | 第0行的首地址 | `int *`（元素指针） | 1000 |
| `a[1]` | 第1行的首地址 | `int *`（元素指针） | 1012 |
| `&a[0][0]` | 第0行第0列元素的地址 | `int *` | 1000 |
| `a[0][0]` | 第0行第0列的元素 | `int` | 1 |

**关键区别**：
- `a` 是**行指针**，类型是 `int(*)[3]`，每次 `+1` 跳过**整行**（3个int，12字节）
- `a[0]` 是**元素指针**，类型是 `int*`，每次 `+1` 跳过**一个元素**（4字节）

**图解**：
```
        a + 0         a + 1
           ↓             ↓
        +-----+-----+-----+-----+-----+-----+
a[0] →  |  1  |  2  |  3  |  4  |  5  |  6  |  ← a[1]
        +-----+-----+-----+-----+-----+-----+
        ↑     ↑     ↑
     a[0]+0 a[0]+1 a[0]+2
```

---

### 3. 等价表达式总结

**访问元素 `a[i][j]` 的等价方式**：

| 方式 | 说明 |
|------|------|
| `a[i][j]` | 最直观的下标访问 |
| `*(a[i] + j)` | 第i行首地址 + j 偏移，再解引用 |
| `*(*(a + i) + j)` | 先跳到第i行，再偏移j列，解引用 |
| `*( &a[0][0] + 3*i + j )` | 按一维数组方式计算 |

**推导过程**：
```
a[i][j]
↓
*(a[i] + j)           // a[i] 是第i行首地址
↓
*(*(a + i) + j)       // a[i] 等价于 *(a + i)
```

## 二、数组指针（行指针）

### 1. 什么是数组指针？

**数组指针**：指向一个数组的指针（指向整行）

```c
int (*pa)[3];  // pa 是一个指针，指向"包含3个int元素的数组"
```

**⚠️ 注意区别**：

| 声明 | 含义 | 类型 |
|------|------|------|
| `int (*pa)[3];` | **数组指针**，指向含3个int的数组 | `int (*)[3]` |
| `int *pa[3];` | **指针数组**，包含3个int指针的数组 | `int *[3]` |

**记忆口诀**：
- `()` 先和 `*` 结合 → **指针**（数组指针）
- `[]` 先和 `pa` 结合 → **数组**（指针数组）

---

### 2. 数组指针与二维数组

```c
int a[2][3];
int (*pa)[3] = a;  // pa 指向二维数组 a
```

**为什么可以赋值？**
- `a` 的类型是 `int (*)[3]`（指向含3个int数组的指针）
- `pa` 的类型也是 `int (*)[3]`
- 类型匹配，可以赋值

**内存图解**：
```
        pa + 0        pa + 1
           ↓             ↓
        +-----+-----+-----+-----+-----+-----+
pa[0] → |  1  |  2  |  3  |  4  |  5  |  6  |  ← pa[1]
        +-----+-----+-----+-----+-----+-----+
```

---

### 3. 用数组指针访问元素

**关键公式**：
```
pa[i][j] 等价于 *(*(pa + i) + j)
```

**推导**：
```
pa[i]        // 第i行的首地址（类型是 int*）
pa[i][j]     // 第i行第j列的元素
*(*(pa + i) + j)  // 等价写法
```

## 三、你的错题详解

### 题目
```c
int a[2][3];
int (*pa)[3] = a;
```

以下哪个是错误的？

A. `x = *(a[0] + 2);`
B. `x = *pa[2];`
C. `x = pa[0][0];`
D. `x = *(pa[1] + 2);`

---

### 逐项分析

#### A. `x = *(a[0] + 2);` ✅ 正确

**分析**：
```
a[0]        // 第0行的首地址，类型 int*
a[0] + 2    // 第0行首地址 + 2个元素，指向 a[0][2]
*(a[0] + 2) // 解引用，得到 a[0][2] 的值
```

**等价于**：`x = a[0][2];`

---

#### B. `x = *pa[2];` ❌ 错误（越界）

**分析**：
```
pa[2]       // 第2行的首地址
```

**问题**：
- 数组 `a` 只有 **2 行**（行号 0 和 1）
- `pa[2]` 访问的是**第2行**，越界了！
- 行号范围：0, 1，没有 2

**图解**：
```
        pa[0]         pa[1]         pa[2] ← 越界！
           ↓             ↓             ↓
        +-----+-----+-----+-----+-----+-----+-----+-----+
        |  1  |  2  |  3  |  4  |  5  |  6  |  ?  |  ?  |
        +-----+-----+-----+-----+-----+-----+-----+-----+
```

---

#### C. `x = pa[0][0];` ✅ 正确

**分析**：
```
pa[0]       // 第0行的首地址
pa[0][0]    // 第0行第0列的元素
```

**等价于**：`x = a[0][0];`

---

#### D. `x = *(pa[1] + 2);` ✅ 正确

**分析**：
```
pa[1]       // 第1行的首地址，类型 int*
pa[1] + 2   // 第1行首地址 + 2个元素，指向 pa[1][2]
*(pa[1] + 2) // 解引用，得到 pa[1][2] 的值
```

**等价于**：`x = pa[1][2];` 或 `x = a[1][2];`

## 四、核心知识点总结

### 1. 二维数组名的类型

```c
int a[2][3];
```

| 表达式 | 类型 | 含义 |
|--------|------|------|
| `a` | `int (*)[3]` | 行指针，+1跳一行 |
| `a[0]` | `int *` | 元素指针，+1跳一个元素 |
| `a[1]` | `int *` | 元素指针，+1跳一个元素 |

### 2. 指针类型必须匹配

```c
int a[2][3];
int (*p1)[3] = a;     // ✓ 正确，类型匹配
int *p2 = a;          // ✗ 错误，类型不匹配
int *p3 = a[0];       // ✓ 正确，a[0] 是 int*
int *p4 = &a[0][0];   // ✓ 正确，元素地址
```

### 3. 访问元素的等价写法

| 写法 | 说明 |
|------|------|
| `a[i][j]` | 下标访问 |
| `*(a[i] + j)` | 行首址+偏移 |
| `*(*(a + i) + j)` | 二级指针访问 |
| `pa[i][j]` | 数组指针访问（pa = a时） |

## 五、常见错误类型

### 错误1：混淆行指针和元素指针

```c
int a[2][3];
int *p = a;        // ✗ 错误！a 是 int(*)[3]，不是 int*
int (*p)[3] = a;   // ✓ 正确
```

### 错误2：越界访问

```c
int a[2][3];
// 只有 a[0] 和 a[1]，没有 a[2]
x = a[2][0];       // ✗ 越界！
```

### 错误3：混淆数组指针和指针数组

```c
int (*p)[3];       // 数组指针，一个指针
int *p[3];         // 指针数组，三个指针
```

---

## 六、判断技巧

### 如何判断 `pa[i]` 是否越界？

1. 看 `pa` 指向的数组有几行
2. 行号范围：0 ~ 行数-1
3. `a[2][3]` 有2行，行号只能是 0 和 1

### 如何判断 `pa[i][j]` 是否越界？

1. 行：0 ~ 行数-1
2. 列：0 ~ 列数-1
3. `a[2][3]`：行号 0,1；列号 0,1,2

## 七、练习题

判断以下代码是否正确：

```c
int a[3][4];
int (*p)[4] = a;

// 1.
p[2][3]

// 2.
p[3][0]

// 3.
*(p[0] + 5)

// 4.
*(*p + 2)
```

<details>
<summary>点击查看答案</summary>

1. **正确**：p[2][3] 是第2行第3列，行列都在范围内
2. **错误**：p[3][0] 行越界（只有0,1,2三行）
3. **正确**：p[0]+5 等价于 &a[0][5]，即 a[1][1] 的位置
4. **正确**：*p 是 p[0]，*p + 2 是 &a[0][2]，*(*p+2) 是 a[0][2]

</details>
