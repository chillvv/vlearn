📘 AI Tutor 前端 UI 重构设计白皮书 (Handover Spec)
1. 产品愿景与定位 (Product Vision)
   定位： 并非简单的“错题记录本”，而是一个 “以知识图谱为核心的 AI 提分引擎”。
   核心逻辑： 翻转传统的“错题中心”逻辑。知识点/题型套路是骨架（主干），错题是诊断样本（果实）。 通过“记录反思 -> 知识点可视化 -> AI 靶向特训”形成闭环。
2. 核心设计哲学 (Design Philosophy)
   方法论重于题目：标签不再是“粗心/审题错误”等废话，必须是**“原子化的题型套路”**（例：#分离参数法、#端点效应）。
   层级化防混乱：采用“学科 > 模块 > 章节 > 题型标签”的三级树状结构，确保海量数据下依然清晰。
   Bento Box (便当盒) 布局：采用类似 Apple/Linear 的模块化卡片布局，信息密度高且极具高级感。
   数据驱动视觉：用颜色的深浅和数字徽章（Badge）直接表达掌握程度，红色代表“薄弱”，绿色代表“掌握”。
3. 全局视觉规范 (Visual System)
   设计风格：Modern SaaS (参考 Linear, Notion, Vercel)。
   配色方案：
   主色：高级克莱因蓝/深蓝色 (#2563EB)。
   背景：极光灰白 (#F8FAFC)。
   卡片：纯白，16px 圆角，超柔和宽泛阴影。
   排版：极致的留白（Whitespace），严谨的对齐，数学公式渲染（KaTeX/MathJax）必须与行高完美契合。
4. 关键 UI 组件详细定义 (Component Specs)
   A. 全局侧边栏 (Fixed Sidebar)
   布局：左侧固定 240px-260px 宽度。
   导航项 (5项)：仪表盘、练习、上传、错题记录本 (Mistake Notebook)、复习中心。
   交互：选中项需有浅蓝色背景、蓝色文字及左侧 4px 宽的蓝色指示条。
   B. 知识宇宙看板 (The Knowledge Universe Grid)
   顶部科目切换 (Subject Switcher)：横向分段控件（Segmented Controls），一次只显示一个科目的数据（降低认知噪音）。
   英雄卡片 (Hero Analytics)：包含一个环形进度条 (Circular Progress) 展示总掌握度。
   GitHub 式热力图 (Activity Heatmap)：
   极致精细度：方块必须微小且精致（建议 10x10px 或 12x12px），间距 2px。
   颜色：使用主色蓝的 5 阶梯度渐变。拒绝巨大的马赛克方块。
   C. 题型标签组件 (Methodology Tag Component)
   严禁采用：单字符串 标签名 (x个错误)。
   必须采用：Flexbox 复合组件。
   左侧 (Title)：文字部分，如 🔴 # 分离参数法。
   右侧 (Micro-Badge)：一个独立的、高亮颜色的圆形或胶囊气泡，里面仅显示错题数量（如 5）或完成勾选（✓）。
   视觉效果：这种“文字 + 独立数字气泡”的组合是现代 SaaS 的标配，极具专业度。
   D. 知识胶囊页 (The Concept Capsule)
   核心功能：每个知识点有一个独立页面。
   展示逻辑：上方展示用户自主复盘的“锦囊妙计”，下方折叠展示相关的错题案例。
   主按钮 (CTA)：显眼的 🚀 AI 靶向特训，引导用户通过做题将“红色标签”转为“蓝色掌握”。
5. 交互流程闭环 (The Golden Flow)
   上传分析：AI 自动拆解多问大题，定位卡壳的子问题。
   打标复盘：AI 推荐“题型套路标签”，用户必须填写复盘总结后方可存入。
   图谱更新：Dashboard 上的知识树节点实时变红/变大，提示薄弱。
   靶向消除：从知识点进入 AI 特训，连对 3 题后，知识节点变绿，完成掌握。
